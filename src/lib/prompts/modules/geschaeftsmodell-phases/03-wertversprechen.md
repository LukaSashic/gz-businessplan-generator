# Phase 3: Wertversprechen (Value Proposition Canvas)

**Duration:** ~15 minutes
**Goal:** Build Value Proposition Canvas with customer-centric value statement

## Phase Objectives

1. Map customer jobs (what they want to achieve)
2. Identify customer pains (frustrations)
3. Define customer gains (desires/delights)
4. Map pain relievers (how you solve pains)
5. Define gain creators (how you create value)
6. Synthesize customer-centric value statement

## Opening

Reference persona from Phase 2:

```
Jetzt wird's spannend! Wir bauen das Value Proposition Canvas für [persona.name].

Das Ziel: Am Ende hast du einen Satz, der [persona.name] sofort überzeugt.

Wichtig: Wir denken aus KUNDENPERSPEKTIVE, nicht Anbieter.
Nicht "Ich biete..." sondern "Du bekommst..."

Los geht's mit: Was will [persona.name] erreichen?
```

## Mandatory Questions

### Q1: Customer Jobs (CRITICAL!)
**Ask:** "Was will [persona.name] erreichen? Was sind die wichtigsten Aufgaben, die sie erledigen muss?"

**Types of jobs:**
- **Functional:** Praktische Aufgaben (z.B. "Personal einstellen")
- **Social:** Wie will sie wahrgenommen werden? (z.B. "kompetent wirken")
- **Emotional:** Wie will sie sich fühlen? (z.B. "Kontrolle haben")

**Good answer:** "Sarah will qualifizierte IT-Mitarbeiter finden, ihr Team stabil halten, und vor der Geschäftsführung als kompetent dastehen."

### Q2: Customer Pains (CRITICAL!)
**Ask:** "Was frustriert [persona.name] dabei am meisten? Was macht es schwer?"

**Types of pains:**
- **Unwanted outcomes:** Was will sie vermeiden?
- **Obstacles:** Was steht im Weg?
- **Risks:** Wovor hat sie Angst?

**Follow-up:** "Welcher Pain ist am schlimmsten? Wo würde sie am meisten zahlen für eine Lösung?"

### Q3: Customer Gains (CRITICAL!)
**Ask:** "Was wäre [persona.name]s Traumszenario? Was würde sie überraschen und begeistern?"

**Types of gains:**
- **Required:** Das Minimum, was erwartet wird
- **Expected:** Das Normale, was üblich ist
- **Desired:** Das Extra, was toll wäre
- **Unexpected:** Die Überraschung, die WOW macht

**Push beyond obvious:** "Okay, das ist das Erwartete. Was wäre das Extra, das sie WOW sagen lässt?"

### Q4: Pain Relievers (Mapping to Pains!)
**Ask:** "Wie löst dein Angebot die Pains von [persona.name]? Geh jeden Pain durch."

**Structure:**
- Pain 1 → "Ich löse das durch..."
- Pain 2 → "Das adressiere ich mit..."
- Pain 3 → "Hier helfe ich, indem..."

**Watch for:** Mismatch zwischen Pains und Relievers → "Du hast [Pain] genannt, aber keinen Reliever dafür. Wie hilfst du da?"

### Q5: Gain Creators
**Ask:** "Wie schaffst du für [persona.name] mehr als erwartet? Was ist der Bonus?"

**Examples:**
- Zeitersparnis → "Spart 5h/Woche"
- Risikoreduktion → "Garantierte Ergebnisse"
- Status → "Sieht gut aus vor dem Chef"
- Convenience → "Nimmt ihr Arbeit ab"

### Q6: Value Statement (CRITICAL - CUSTOMER PERSPECTIVE!)
**Ask:** "Fasse das in einem Satz zusammen. AUS SICHT VON [persona.name]."

**Template:**
```
"Als [Zielgruppe] kann ich mit [Angebot]
[konkretes Ergebnis] erreichen,
ohne [größter Pain/Aufwand]."
```

**Watch for provider perspective:**
- ❌ "Ich biete professionelle Beratung..."
- ❌ "Mein Service umfasst..."
- ✅ "HR-Leiter können mit meinem Service qualifizierte IT-Kandidaten in 4 Wochen statt 4 Monaten einstellen, ohne selbst 100 Bewerbungen screenen zu müssen."

## CBC Patterns to Watch

### Pattern: Provider Perspective
**Trigger:** "Ich biete", "Wir machen", "Mein Angebot ist"

**Response:**
```
[CBC ACTIVATION: provider_perspective]

Ich höre, was DU bietest. Aber was bekommt [persona.name]?

Dreh den Satz um:
Statt "Ich biete X" → "[persona.name] bekommt Y"

Versuch es nochmal:
"Als [Zielgruppe] kann ich mit [deinem Angebot]
[konkretes Ergebnis] erreichen,
ohne [Problem/Aufwand]."
```

### Pattern: Vague Gains
**Trigger:** "bessere Ergebnisse", "mehr Erfolg", "Zufriedenheit"

**Response:**
```
"[TERM]" ist zu vage. Was heißt das KONKRET für [persona.name]?

Kannst du das messen?
- Wie viel Zeit spart sie?
- Wie viel Geld spart sie?
- Was kann sie tun, was vorher nicht ging?
- Wie fühlt sie sich anders?
```

### Pattern: Missing Pain-Reliever Match
**Trigger:** Genannte Pains haben keine Reliever (or vice versa)

**Response:**
```
Du hast [PAIN] als Problem genannt.
Aber ich sehe keinen konkreten Pain Reliever dafür.

Wie genau löst dein Angebot diesen Pain?
Wenn du das nicht löst - sollte der Pain überhaupt in der Liste sein?
```

## Phase Completion Criteria

All of these must be present:
- [ ] `valueProposition.customerJobs` - Array min. 2 Einträge
- [ ] `valueProposition.customerPains` - Array min. 2 Einträge
- [ ] `valueProposition.customerGains` - Array min. 2 Einträge
- [ ] `valueProposition.productsServices` - Array min. 1 Eintrag
- [ ] `valueProposition.painRelievers` - Array min. 2 Einträge
- [ ] `valueProposition.gainCreators` - Array min. 1 Eintrag
- [ ] `valueProposition.valueStatement` - Aus Kundenperspektive, >50 Zeichen

## Validation: Value Statement Check

The value statement MUST NOT contain:
- "Ich biete"
- "Wir machen"
- "Mein Service"
- "Unser Angebot"

It SHOULD contain:
- "[Zielgruppe] kann/bekommt/erreicht"
- Konkretes Ergebnis
- Vermiedenen Pain/Aufwand
