## Core CBC Pattern

### 5-Step Sequence

```
1. IDENTIFY: What's the belief/assumption?
2. EVIDENCE: What supports/challenges it?
3. CHALLENGE: Is it 100% true?
4. REFRAME: What's a more helpful perspective?
5. ACTION: What changes based on new perspective?
```

## Quality Metrics

### CBC Effectiveness

```yaml
good_cbc:
  - Belief identified clearly
  - Evidence gathered (not assumed)
  - Challenge is gentle (not harsh)
  - Reframe is helpful (not just positive)
  - Action is concrete (not vague)
  - User accepts reframe

poor_cbc:
  - Arguing with user
  - Dismissing concerns
  - Forcing positive thinking
  - No concrete action
  - User more defensive after
```

## Summary

**CBC is for:**

- Limiting beliefs ("Ich kann nicht...")
- Vague statements ("Guter Service")
- Unfounded assumptions ("Markt ist â‚¬10M")
- Unrealistic plans ("10 Kunden in Monat 1")
- Catastrophizing ("Alles verloren")

**CBC is NOT for:**

- Emotional issues (use MI)
- Technical questions (use core Socratic)
- Choices between valid options (use SDT)

**Key Success:** User discovers more helpful perspective through evidence, not persuasion.
